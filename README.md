## 开发模式

### 🚀 快速启动（推荐）

**一键启动前后端服务**
```bash
cd server
npm run dev-all
```

这个命令会同时启动：
- 后端Node.js服务器（端口3678）
- React前端开发服务器（端口3001）

### 🔧 分别启动服务

**方式一：使用两个终端窗口**
```bash
# 终端1：启动后端服务器
cd server
npm run dev

# 终端2：启动React前端开发服务器
cd server
npm run dev-client
```

**方式二：后台启动后端**
```bash
# 后台启动后端服务
cd server
npm run dev &

# 启动前端开发服务器
npm run dev-client
```

### 📋 可用的npm脚本

| 命令 | 功能 | 说明 |
|------|------|------|
| `npm start` | 生产模式启动 | 启动生产版本服务器 |
| `npm run dev` | 开发模式后端 | 使用nodemon启动后端，支持热重载 |
| `npm run dev-client` | 开发模式前端 | 启动Webpack开发服务器 |
| `npm run dev-all` | 开发模式全栈 | 同时启动前后端服务 |
| `npm run build` | 构建生产版本 | 打包React应用到public/dist |

### ⚙️ 服务器配置说明

**后端服务器（端口3678）**
- 提供API接口和WebSocket服务
- 支持文件上传功能
- 自动连接MongoDB（失败时使用内存存储）
- 支持跨域请求

**前端开发服务器（端口3001）**
- Webpack开发服务器，支持热重载
- 自动代理API请求到后端服务器
- 支持React组件的实时更新
- 包含开发调试工具

### 🌐 代理配置

前端开发服务器已配置代理，以下路径会自动转发到后端：
- `/socket.io` → `http://localhost:3678`
- `/api` → `http://localhost:3678`
- `/uploads` → `http://localhost:3678`

### 🔍 启动状态检查

**检查服务是否正常运行：**
```bash
# 检查端口占用
lsof -i :3678  # 后端服务
lsof -i :3001  # 前端服务

# 测试API连接
curl http://localhost:3678/api/health

# 查看服务日志
npm run dev  # 查看后端日志
```

### ⚠️ 常见启动问题

1. **端口被占用**
   ```bash
   # 杀死占用端口的进程
   kill -9 $(lsof -ti:3678)
   kill -9 $(lsof -ti:3001)
   ```

2. **依赖安装问题**
   ```bash
   # 清除缓存重新安装
   rm -rf node_modules package-lock.json
   npm install
   ```

3. **MongoDB连接失败**
   - 应用会自动切换到内存存储模式
   - 数据在服务器重启后会丢失
   - 生产环境建议配置MongoDB连接

## 开发说明

### 核心功能详解

#### 1. 消息历史记录
- 用户加入聊天室时，自动接收最近50条消息
- 支持MongoDB和内存两种存储方案
- 消息包含完整信息：用户、内容、时间戳、类型等

#### 2. 文件传输系统
- **单文件下载**：直接浏览器下载，支持大文件流式传输
- **文件夹下载**：自动压缩成ZIP文件，递归包含子目录
- **外接驱动器支持**：识别和列出所有已挂载的磁盘
- **进度显示**：文件下载进度、速度实时显示

#### 3. 媒体上传
- **文件上传**：支持通过上传按钮选择文件
- **剪贴板粘贴**：支持Ctrl+V (Win) / Cmd+V (Mac) 粘贴图片
- **拖拽上传**：支持拖拽文件到输入框上传
- **中文文件名**：自动处理UTF-8编码，正确显示中文文件名

#### 4. 文件浏览器
- **磁盘导航**：显示系统磁盘和外接驱动器
- **路径操作**：支持进入目录、返回上级目录
- **根目录识别**：自动识别各平台根目录（/ 在macOS/Linux，C:\ 在Windows）
- **下载操作**：直接从浏览器下载单个文件或整个目录

#### 5. 数据持久化
```
存储优先级：
1. 首先尝试连接MongoDB - 消息和用户数据持久化
2. 若MongoDB失败 → 自动切换到内存存储
   - 用户数据保存到内存Map
   - 消息保存到数组（最多500条）
   - 服务器重启后数据丢失
```

#### 6. 消息类型
- **文本消息**：支持最多500字符
- **图片消息**：支持预览和点击放大
- **文件消息**：显示文件大小、MIME类型、下载链接
- **系统消息**：用户加入/离开提示

## 故障排除

### 常见问题

1. **端口冲突**
   - 确保3678端口（后端）和3001端口（前端）未被占用

2. **WebSocket连接失败**
   - 检查后端服务器是否正常运行
   - 确认防火墙设置

3. **构建失败**
   - 清除node_modules重新安装依赖
   - 检查Node.js版本（推荐16+）

4. **消息历史不显示**
   - 确保MongoDB服务正常运行或接受内存存储模式
   - 检查后端日志是否有消息保存错误
   - 历史消息最多保留50条

5. **文件上传/下载失败**
   - 确保uploads目录有写权限：`chmod 755 uploads`
   - 检查磁盘空间是否充足
   - 对于外接驱动器，确保已正确挂载

6. **中文文件名显示乱码**
   - 此问题已修复，后端自动处理latin1→UTF-8转码
   - 如仍有问题，确保MongoDB字符集为UTF-8

7. **剪贴板粘贴不工作**
   - 仅在HTTPS或localhost环境下有效
   - 某些浏览器可能需要授予权限
   - 检查浏览器控制台是否有权限错误

### 调试模式

在浏览器开发者工具中可以查看：
- WebSocket连接状态和消息
- Socket.IO事件日志
- 文件上传/下载进度
- React组件状态和性能
- 网络请求（API、文件传输）

### 性能优化建议

1. **消息加载**
   - 历史消息限制为50条，防止初始化缓慢
   - 可根据需要在config中调整限制

2. **文件传输**
   - 大文件使用流式传输，避免内存溢出
   - 文件夹下载自动压缩，减少带宽

3. **内存存储**
   - 消息最多保留500条，自动淘汰旧消息
   - 生产环境强烈建议使用MongoDB
